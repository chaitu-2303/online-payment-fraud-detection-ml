## Development Commands

**Running the Application:**
```bash
# Start the development server
python -m uvicorn app.main:app --reload

# Or run directly
python app/main.py
```

**Dependencies:**
```bash
# Install requirements
pip install -r requirements.txt
```

**Testing:**
```bash
# Run authentication tests
python app/scripts/test_create_user.py
python app/scripts/test_register_post.py
```

## Project Structure

**Core Components:**
- `app/main.py` - FastAPI application entry point
- `app/routes/` - Route handlers for pages, auth, and admin functionality
- `app/services/` - Business logic (authentication, ML inference, tokens)
- `app/templates/` - Jinja2 HTML templates
- `app/static/` - CSS and static assets
- `ml/` - Machine learning components
  - `ml/artifacts/` - Trained model pipeline and threshold
  - `ml/train.py` - Model training script
- `app/data/` - Data files (CSV datasets, SQLite user database)

**Key Files:**
- `app/routes/pages.py` - Main dashboard and prediction routes
- `app/routes/auth.py` - User authentication (login, register, logout, admin)
- `app/services/inference.py` - ML model inference service
- `app/services/auth_db.py` - SQLite-based user authentication
- `app/routes/admin.py` - Admin user management

## Architecture Overview

This is a web-based fraud detection system with:

1. **Frontend**: Server-rendered HTML using Jinja2 templates
2. **Backend**: FastAPI application with session-based authentication
3. **ML Component**: Pre-trained scikit-learn pipeline for fraud prediction
4. **Data Layer**: 
   - SQLite database for user management (`app/data/users.db`)
   - CSV file for transaction data analysis (`data/onlinefraud.csv`)
5. **Authentication**: Custom session management with remember-me tokens

## Common Tasks

**Adding New Routes:**
1. Create router in `app/routes/`
2. Import and include in `app/main.py`
3. Use existing templates or create new ones in `app/templates/`

**Modifying ML Features:**
1. Update feature processing in `app/routes/pages.py` (predict endpoint)
2. Retrain model using `ml/train.py` if needed
3. Update pipeline in `ml/artifacts/`

**Extending Authentication:**
1. Modify `app/services/auth_db.py` for schema changes
2. Update `app/routes/auth.py` for new endpoints
3. Adjust session middleware settings in `app/main.py`

**Static Assets:**
- CSS files in `app/static/`
- Reference in templates using `/static/` path

## Environment Configuration

Set environment variable for secret key:
```bash
set APP_SECRET_KEY=your-secret-key-here
```
(Defaults to "CHANGE_THIS_SECRET_KEY" for development)

## ML Artifacts

The system expects these files in `ml/artifacts/`:
- `pipeline.pkl` - Trained scikit-learn pipeline
- `threshold.json` - Classification threshold configuration

These are generated by running `ml/train.py` (not included in repo).
